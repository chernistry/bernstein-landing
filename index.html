How it works

Three steps from goal to shipped code. No babysitting.

1

Set a goal

Describe what you want in plain English. Bernstein decomposes it into tasks with the right roles, models, and dependencies.

2

Agents work in parallel

Each agent gets its own git worktree. No file conflicts. Opus for architecture, Sonnet for implementation, Haiku for tests — automatically routed.

3

Verified and merged

Quality gates run lint, types, and tests on every result. Only verified work gets merged. Failed tasks retry with escalated models.

Built for real engineering teams

Not a demo. A production orchestration system.

🎯

Deterministic scheduling

The orchestrator is pure Python. Zero LLM tokens wasted on coordination. Predictable, debuggable, fast.

🔌

29 agent adapters

Claude Code, Codex, Gemini CLI, Aider, Amp, Cursor, and 23 more. Mix models in one run. Switch providers without changing anything.

🛡️

Quality gates

Lint, type check, tests, security scan, architecture conformance — all run automatically before merge. Failed work gets retried.

💰

Cost-aware routing

Epsilon-greedy bandit learns which model works best per task type. Save 50-60% vs uniform expensive models.

🌳

Git worktree isolation

Each agent works in its own worktree. No merge conflicts between agents. Clean commit history.

📊

Full observability

Cost tracking, token monitoring, quality trends, post-mortem reports. Know exactly what happened and how much it cost.
